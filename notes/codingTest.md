# 코테 문제 유형 파악 (요즘 5문제 코테 실전판)

목표: 문제 읽자마자 “아 이거는 이거 쓰라는 문제구나” 감을 만드는 체크리스트.

---

## 0) 10초 루틴 (이거만 습관화)
1) **결과가 뭐냐**: 최단/최대/개수/가능여부/정답값의 최소(최대)
2) **제한(N, Q)** 보고 복잡도 컷
   - N ~ 1e3: O(N^2) 가능
   - N ~ 2e5: O(N log N) / O(N) 강제
   - Q(질의) ~ 1e5+: “쿼리 자료구조” 냄새 (Fenwick/SegTree/PrefixSum)
   - N이 작다(<= 15~20): 백트래킹/부분집합/비트마스크 가능
3) **데이터 형태**: 배열/문자열/격자/그래프/트리/질의형
4) 아래 키워드 매칭표로 확정

---

## 1) 요즘 5문제 코테에서 “진짜 자주” 나오는 유형 TOP 12

### 1. 해시/카운팅 (Map/Set)
- 키워드: 중복, 등장 횟수, 서로 다른 개수, 아나그램, 두 수의 합, 포함 여부
- 도구: HashMap/HashSet

### 2. 정렬 + 한 번 훑기 (스캔/그리디 느낌)
- 키워드: 정렬 후, 최소/최대, 최대로 많이 선택, 겹침/끝나는 시간
- 도구: sort + loop (+ 우선순위큐가 붙기도 함)

### 3. BFS/DFS (격자/그래프 기본 탐색)
- 키워드: 연결 요소 개수, 섬 개수, 도달 가능, 미로, 최단 “횟수”
- 도구: BFS/DFS
- 팁: 비용이 전부 1이면 BFS가 최단거리

### 4. 최단거리 (가중치) = 다익스트라
- 키워드: 비용/가중치/최소 비용 경로 (가중치 >= 0)
- 도구: Dijkstra + PriorityQueue

### 5. DP (기본 1D/2D)
- 키워드: 경우의 수(mod), 최대/최소, 만들기/변환, 선택을 반복
- 도구: DP(상태 정의가 핵심)

### 6. 백트래킹 (요즘 매우 자주)
- 키워드: 모든 경우/조합/순열/배치, N이 작음, 조건 만족 찾기, 최솟값 갱신
- 도구: DFS로 “선택/미선택” + 가지치기(pruning)
- 확정 냄새(2개 이상이면 거의 백트래킹):
  - N <= 15~20 (혹은 선택 횟수 <= 10)
  - 방문/사용 여부가 상태임
  - 중간에 불가능하면 자르는 조건이 있음

### 7. 투포인터/슬라이딩 윈도우
- 키워드: 연속 구간, 부분배열(연속), 길이 최소/최대, 합이 K 이상/이하
- 팁: 원소가 비음수(양수 포함)일 때 특히 잘 먹힘

### 8. 누적합(prefix sum) + 해시/정렬
- 키워드: 구간합이 K, 부분배열 합의 개수, 구간 조건 만족 개수
- 도구: PrefixSum + HashMap(카운트) / PrefixSum + 정렬/이분

### 9. 이분탐색(파라메트릭)
- 키워드: 최소의 최대 / 최대의 최소 / 최소 시간·용량·거리
- 확정 조건: 어떤 x에 대해 check(x)로 “가능/불가능” 판정되고 단조성 있음

### 10. 힙(우선순위큐)
- 키워드: 매번 최댓값/최솟값, K번째, top-K 유지, 스케줄링
- 도구: PriorityQueue

### 11. 구간쿼리 + 업데이트 = Fenwick/SegTree
- 키워드: Q가 큼 + 값 변경(update) 있음 + 구간 합/최대/최소 반복
- 도구: Fenwick(합), SegTree(합/최대/최소)

### 12. DSU(유니온파인드)
- 키워드: 합치기, 같은 그룹, 연결 여부, 간선 추가하면서 질의
- 도구: Union-Find
